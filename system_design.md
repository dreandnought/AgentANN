# 多智能体BP神经网络系统设计大纲 (Multi-Agent BP Neural Network)

## 一、 系统架构理念
本系统旨在将传统的前向传播和反向传播（BP）算法解耦到分布式的多智能体（Multi-Agent）系统中。我们将**一个神经元视为一个独立的Agent**。通过异步消息传递（Asyncio Message Passing），智能体之间不直接共享内存，而是通过“发消息”来完成前向计算与梯度回传。

## 二、 网络拓扑设计 (2层、8个隐藏神经元)
对于鸢尾花分类（Iris）任务（输入维度4，输出类别3），系统拓扑如下：

1. **输入层 (Input Layer)**：无独立Agent。由全局 `Coordinator`（协调器）代理，负责读取 4 维的特征向量，并广播给隐藏层的所有Agent。
2. **隐藏层 (Hidden Layer)**：包含 **8个 `NeuronAgent` (H0-H7)**。每个Agent独立维护自己与输入层之间连接的 4 个权重（Weights）和 1 个偏置（Bias）。
3. **输出层 (Output Layer)**：包含 **3个 `NeuronAgent` (O0-O2)**（分别代表类别 0-2）。每个Agent独立维护自己与隐藏层之间连接的 8 个权重和 1 个偏置。

总计只需要 **11个核心Agent** 即可完成网络的分布式模拟。

## 三、 Agent 状态与交互机制

每个 `NeuronAgent` 拥有独立的邮箱（Queue）和事件循环。

### 1. Agent 的内部私有状态 (State)
- `weights`: 权重向量。
- `bias`: 偏置标量。
- `activation`: 激活函数（隐藏层 ReLU，输出层 Sigmoid）。
- `cache`: 缓存当前前向传播的输入，用于计算反向传播时的梯度。

### 2. 前向传播交互 (Forward Pass)
- **协调器**将样本数据 $X$ 发送给隐藏层的 8 个 Agent。
- **隐藏层 Agent** 收到 $X$ 后，计算加权和 $z = W \cdot X + b$ 并激活 $a = \sigma(z)$。然后将标量 $a$ 发送给所有的输出层 Agent。
- **输出层 Agent** 挂起等待，收集齐隐藏层 8 个 Agent 发来的 $a$，拼凑成完整的输入向量，计算自己的输出 $y$，并发送回协调器。

### 3. 反向传播交互 (Backward Pass)
- **协调器**对比预测结果与真实标签，计算误差梯度，并将误差信号发给对应的输出层 Agent。
- **输出层 Agent** 收到误差后：
  1. 计算并更新自己的 $W$ 和 $b$。
  2. 计算关于其输入的梯度，并将这部分梯度精确发回给对应的隐藏层 Agent。
- **隐藏层 Agent** 挂起等待，收集齐输出层 3 个 Agent 发来的梯度并求和后：
  1. 计算并更新自己的 $W$ 和 $b$。

## 四、 全局状态管理与独立日志
1. **全局权重同步 (`global_weights.json`)**：
   Agent 每次完成参数更新后，发送特殊的 `WEIGHT_UPDATE` 消息给 Coordinator。Coordinator 汇总后统一覆写该文件，避免并发文件锁竞争。
2. **独立日志追踪 (`logs/` 目录)**：
   每个 Agent 初始化专属的 `logging.Logger`（如 `logs/H3.log`）。独立记录每次前向接收的输入、激活值，以及反向收到的梯度和参数更新详情。
