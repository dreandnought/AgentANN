# 换脑改造方案：将 NeuronAgent 迁移至大模型驱动

## 一、 背景与目的
当前系统 (`src/core/neuron_agent.py`) 的前向与反向传播完全由本地 CPU (Numpy) 执行，这主要用于验证系统通信框架、并发调度与容错机制的正确性。
在系统测试稳定后，我们的最终目标是将神经网络的节点“拟人化”，即**将本地数值计算替换为大语言模型（LLM）的推理过程**，从而观察大模型是否具备执行微积分链式法则与矩阵运算的“涌现能力”。

## 二、 改造点分析
我们需要对 `NeuronAgent` 进行“换脑手术”，核心改造集中在以下两个方法：
1. `_handle_forward(msg)`: 前向传播
2. `_handle_backward(msg)`: 反向传播

### 1. 引入 LLM 客户端
在 `NeuronAgent.__init__` 中注入 `LLMClient` 实例。

### 2. 前向传播 (Forward) 的 LLM 改造
**原逻辑**：接收输入 $X$ -> 执行 `np.dot(W, X) + b` -> 激活函数 -> 发送给下游。
**新逻辑**：
- **Prompt 组装**：提取当前 Agent 的参数 $W, b$，以及接收到的输入向量 $X$。
- **调用 LLM**：使用 System Prompt 明确当前节点是隐藏层还是输出层，并要求它执行加权和及激活操作。
- **强制 JSON 输出**：要求 LLM 仅输出 JSON 格式的结果（例如：`{"activation_value": 0.85}`）。
- **结果解析与校验**：解析 LLM 吐出的 JSON，若非数字或格式错误则触发重试。成功后，将其作为结果广播给下游。

### 3. 反向传播 (Backward) 的 LLM 改造
**原逻辑**：接收下游梯度/误差 -> 依据数学求导公式计算 $\delta, dW, db$ -> 更新参数 -> 传梯度给上游。
**新逻辑**：
- **Prompt 组装**：传入上一步缓存的输入 $X$、当前的 $W, b$、激活函数类型，以及本次收到的误差信号/梯度信号。
- **要求推导**：命令 LLM 依据链式法则，逐步写出求导过程（可选），但必须在 JSON 块中返回最终的梯度。
- **强制 JSON 输出**：
  ```json
  {
    "dW": [0.01, -0.05, ...],
    "db": 0.02,
    "grad_to_upstream": [0.1, 0.2, ...] // 仅输出层需要传给隐藏层
  }
  ```
- **解析与更新**：解析 LLM 返回的 JSON，在本地使用学习率进行参数更新 `W = W - lr * dW`。

## 三、 容错与成本控制策略
1. **JSON 结构校验**：LLM 存在幻觉，必须引入 Pydantic 或 JSON Schema 严格校验返回的数据维度是否与 $W$ 的维度一致。
2. **自动重试机制 (Retry)**：若解析失败或数值溢出，向 LLM 追加一条 `UserMessage` 报错并要求其修正，最多重试 3 次。
3. **降级机制 (Fallback)**：若重试耗尽仍失败，可考虑由本地 Numpy 接管本次计算，并记录死信日志 `logs/dead_letter.log` 以防整个网络死锁。
4. **Token 限制**：鸢尾花数据集的4维输入不存在长文本问题，但需要保证模型能准确进行浮点数计算。
